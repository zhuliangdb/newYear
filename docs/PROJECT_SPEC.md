# 新年主题网站项目规范

## 1. 项目目标与范围

### 1.1 项目目标

- 开发一个融合安徽、河南地域文化特色的新年主题网站，打造独特的新年盛宴
- 提供丰富的互动功能，包括新年倒计时、愿望墙、留言板、烟花效果等
- 展示安徽、河南地区的传统新年文化，促进地域文化的传播与交流
- 采用现代化的前端技术，打造美观、流畅、响应式的用户体验
- 确保网站在不同设备上的良好显示效果

### 1.2 项目范围

#### 1.2.1 核心功能

- **首页**：英雄区域、文化特色预览、核心功能展示、活动预览、用户评价、页脚
- **文化特色**：安徽黄梅戏、徽州剪纸、河南豫剧脸谱、朱仙镇年画等地域文化展示
- **新年活动**：活动列表、活动详情、活动报名
- **新年惊喜**：倒计时、愿望墙、留言板、烟花效果、运势预测
- **关于我们**：项目介绍、团队信息、联系方式

#### 1.2.2 技术范围

- 前端：Vue 3 + Vite + Vue Router + Element Plus
- 部署：GitHub Pages
- 数据库：Firebase Realtime Database

#### 1.2.3 非范围

- 电商功能（如商品购买）
- 用户登录系统（仅使用Firebase匿名认证）
- 支付功能
- 多语言支持（仅支持中文）

## 5. 开发流程规范

### 5.1 分支管理

- **master**：主分支，用于生产环境部署
- **develop**：开发分支，集成所有功能开发
- **feature/xxx**：功能分支，用于开发新功能
- **bugfix/xxx**：修复分支，用于修复bug

### 5.2 开发流程

1. **需求分析**：分析项目需求，确定功能模块
2. **设计阶段**：设计UI界面，确定技术方案
3. **开发阶段**：
   - 从develop分支创建feature分支
   - 实现功能模块
   - 提交代码到feature分支
   - 合并feature分支到develop分支
4. **测试阶段**：
   - 进行功能测试
   - 进行兼容性测试
   - 进行性能测试
5. **部署阶段**：
   - 合并develop分支到master分支
   - GitHub Actions自动部署到GitHub Pages

### 5.3 代码提交规范

#### 5.3.1 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

#### 5.3.2 类型说明

- **feat**：新功能
- **fix**：修复bug
- **docs**：文档更新
- **style**：代码风格调整
- **refactor**：代码重构
- **test**：测试代码
- **chore**：构建过程或辅助工具的变动

#### 5.3.3 示例

```
feat(home): 添加英雄区域动效

添加了灯笼摇摆和烟花效果，提升节日氛围

Closes #1
```

## 6. 代码质量要求

### 6.1 代码风格

- 使用ESLint和Prettier保持代码风格一致
- 缩进使用2个空格
- 字符串使用单引号
- 组件命名使用PascalCase
- 变量命名使用camelCase
- 常量命名使用UPPER_CASE
- 函数命名使用camelCase
- 文件命名：
  - 组件文件：PascalCase.vue
  - 工具文件：kebab-case.js
  - 样式文件：kebab-case.css

### 6.2 代码规范

- 组件化开发，合理拆分组件
- 使用Vue 3组合式API
- 避免使用Vue 2的选项式API
- 使用Pinia进行状态管理，避免直接操作DOM
- 使用Vue Router进行路由管理
- 合理使用computed和watch
- 避免内存泄漏，及时清理定时器和事件监听器
- 代码注释清晰，关键逻辑必须有注释

### 6.3 性能优化

- 组件懒加载
- 图片懒加载
- 合理使用v-if和v-show
- 避免不必要的计算和渲染
- 优化CSS选择器
- 使用CDN加载第三方库
- 合理配置PWA缓存

### 6.4 可访问性

- 使用语义化HTML标签
- 确保颜色对比度符合WCAG标准
- 为图片添加alt属性
- 确保键盘导航正常
- 支持屏幕阅读器

## 8. 部署流程

### 8.1 构建流程

1. **代码检查**：运行ESLint检查代码质量
   ```bash
   npm run lint
   ```

2. **构建项目**：
   ```bash
   npm run build
   ```

3. **构建分析**：
   ```bash
   npm run build -- --report
   ```

### 8.2 部署步骤

1. **配置GitHub Pages**：
   - 登录GitHub，进入项目仓库
   - 点击"Settings" -> "Pages"
   - 在"Build and deployment"部分，选择：
     - Source: Deploy from a branch
     - Branch: gh-pages / (root)
   - 点击"Save"

2. **推送代码**：
   ```bash
   git add .
   git commit -m "Deploy to GitHub Pages"
   git push origin master
   ```

3. **查看部署状态**：
   - 进入项目仓库的"Actions"标签页
   - 查看"Deploy to GitHub Pages"工作流的运行状态
   - 部署成功后，访问 `https://<username>.github.io/<repository-name>/newYear/` 查看网站

### 8.3 自定义域名配置

1. **添加CNAME记录**：
   在域名解析服务商处添加以下CNAME记录：
   | 类型 | 主机记录 | 记录值 | TTL |
   |------|----------|--------|-----|
   | CNAME | your-subdomain | <username>.github.io. | 10分钟 |

2. **配置GitHub Pages**：
   - 进入项目仓库的"Settings" -> "Pages"
   - 在"Custom domain"部分，输入你的自定义域名
   - 点击"Save"

3. **等待DNS生效**：
   DNS记录生效后，访问你的自定义域名即可查看网站

### 8.4 部署监控

- 监控GitHub Actions工作流状态
- 监控网站访问状态
- 监控页面加载速度
- 监控用户反馈

### 11.4 常见问题

#### 11.4.1 页面404错误

- 确保Vue Router使用的是Hash模式
- 检查GitHub Pages的部署分支和目录是否正确
- 等待DNS记录生效

#### 11.4.2 资源加载失败

- 检查vite.config.js中的base配置是否正确
- 确保构建时使用了正确的base URL
- 检查资源路径是否正确

#### 11.4.3 Firebase连接失败

- 检查.env文件中的Firebase配置是否正确
- 确保Firebase项目已启用实时数据库
- 检查Firebase安全规则是否允许访问

#### 11.4.4 构建失败

- 检查Node.js版本是否符合要求
- 确保所有依赖已正确安装
- 检查代码中是否有语法错误
